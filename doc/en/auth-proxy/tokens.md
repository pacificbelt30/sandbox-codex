## Token Format

```
cdx-<64 hex chars>

Example: cdx-a1b2c3d4e5f6789012345678901234567890abcdef1234567890abcdef1234
```

- 32 random bytes generated by `crypto/rand`
- Hex-encoded with `cdx-` prefix
- Total: 68 characters

## Token Lifecycle

```
                 IssueToken()
                      │
                      ▼
              ┌───────────────┐
              │ tokenRecord    │
              │ Token: "cdx-" │
              │ ContainerName │
              │ IssuedAt      │
              │ ExpiresAt     │  ← IssuedAt + TTL (default 3600 s)
              └───────┬───────┘
                      │ stored in memory
                      ▼
              ┌───────────────┐
              │ tokens map    │◀─── RevokeToken() for immediate deletion
              │ [name -> rec] │
              └───────┬───────┘
                      │
               scanned every 30 s
                      │
                      ▼
              expired tokens deleted (expireLoop)
```

### When Tokens are Issued

| Trigger | Method |
|---|---|
| `codex-dock run` execution | `proxy.IssueToken()` called before container start |
| External issuance | Via `POST /admin/issue` endpoint |

### When Tokens are Revoked

| Trigger | Method |
|---|---|
| Container stop | `sandbox.Manager.Stop()` calls `proxy.RevokeToken()` |
| TTL expiry | `expireLoop` removes expired tokens every 30 seconds |
| Explicit revocation | `POST /revoke` or `POST /admin/revoke` |
| Proxy shutdown | `proxy.Stop()` clears all tokens |

### TTL Configuration

| Setting | Default | How to Change |
|---|---|---|
| TTL | 3600 s (1 hour) | `--token-ttl <seconds>` or `config.toml` `default_token_ttl` |

## Environment Variables Injected into Containers

| Variable | Value | Lifecycle |
|---|---|---|
| `CODEX_AUTH_PROXY_URL` | `http://<proxy>:<PORT>` | `unset` by `entrypoint.sh` after use |
| `CODEX_TOKEN` | `cdx-<hex64>` — used for `/token` request | `unset` by `entrypoint.sh` after use |
| `OPENAI_BASE_URL` | `http://<proxy>:<PORT>/v1` | Persists — Codex CLI uses this at runtime |
| `CODEX_REFRESH_TOKEN_URL_OVERRIDE` | `http://<proxy>:<PORT>/oauth/token?cdx=<token>` | OAuth mode only |

In OAuth mode, `entrypoint.sh` also writes these files:

| File | Contents |
|---|---|
| `/home/codex/.codex/auth.json` | `access_token`: placeholder, `id_token`: real JWT, `refresh_token`: empty string |
| `/home/codex/.config/codex/config.toml` | `chatgpt_base_url = "http://<proxy>:<PORT>/chatgpt/"` |

## Credential Priority (API Key Mode)

```
1. OPENAI_API_KEY environment variable
2. ~/.config/codex-dock/apikey  (saved by codex-dock auth set)
3. ~/.codex/auth.json
```

OAuth mode is activated when `~/.codex/auth.json` contains `refresh_token` or `auth_mode: "chatgpt"`.

## Security Considerations

### Implemented Protections

| Protection | Status | Details |
|---|---|---|
| API key isolation | ✅ | Container receives only placeholder; proxy injects real key |
| access_token isolation | ✅ | Real access token never reaches container even in OAuth mode |
| refresh_token protection | ✅ | Never sent to container; proxy handles refresh via `/oauth/token` relay |
| Short-lived tokens | ✅ | TTL-scoped; immediately revoked on container stop |
| API traffic relay | ✅ | `/v1/` and `/chatgpt/` reverse proxies eliminate direct external API access |
| No credential logging | ✅ | Auth info never written to stdout/stderr |
| No auth.json bind-mount | ✅ | Container auth.json contains only a placeholder access_token |

### Known Issues

| ID | Issue | Severity | Details |
|---|---|---|---|
| NF-SEC-01 | Plaintext HTTP | High | TLS/UNIX socket not implemented; designed for Docker internal use only |
| F-AUTH-06 | No container ID verification | Medium | Token tied to container name but not container ID |

## Implementation Reference

```go
// Create Auth Proxy
proxy, _ := authproxy.NewProxy(authproxy.Config{
    TokenTTL:    3600,
    Verbose:     true,
    ListenAddr:  "0.0.0.0:0",
    AdminSecret: "my-secret",
})

proxy.Start()

token, _ := proxy.IssueToken("my-container", 3600)

fmt.Println(proxy.Endpoint())           // "http://127.0.0.1:XXXXX"
fmt.Println(proxy.ContainerEndpoint())  // "http://host.docker.internal:XXXXX"

if proxy.IsOAuthMode() {
    // Set CODEX_REFRESH_TOKEN_URL_OVERRIDE
}

proxy.RevokeToken("my-container")
defer proxy.Stop()
```
